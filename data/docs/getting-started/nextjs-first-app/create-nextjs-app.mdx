## Next.js 앱 개요

React로부터 웹 애플리케이션을 처음부터 구축하려면 고려해야 할 많은 중요한 사항이 있습니다:

- 코드는 webpack과 같은 번들러를 사용하여 번들되어야 하며 Babel과 같은 컴파일러를 사용하여 변환되어야 합니다.
- 코드 분할과 같은 생산 최적화를 수행해야 합니다.
- 성능과 SEO를 위해 몇몇 페이지를 정적으로 사전 렌더링하고 싶을 수 있습니다. 또한 서버 사이드 렌더링 또는 클라이언트 사이드 렌더링을 사용하려고 할 수 있습니다.
- 데이터 스토어에 React 앱을 연결하기 위해 서버 측 코드를 작성해야 할 수 있습니다.

프레임워크는 이러한 문제점을 해결할 수 있습니다. 그러나 이러한 프레임워크는 적절한 추상화 수준을 가져야 합니다 — 그렇지 않으면 매우 유용하지 않을 것입니다. 또한 "개발자 경험"이 탁월해야하며, 코드를 작성하는 동안 당신과 팀이 훌륭한 경험을 할 수 있어야 합니다.

### Next.js: React 프레임워크
Next.js, React 프레임워크를 소개합니다. Next.js는 위의 모든 문제에 대한 해결책을 제공합니다. 그러나 더 중요한 것은, React 애플리케이션을 구축할 때 성공을 위한 기반을 제공합니다.

Next.js는 최고 수준의 개발자 경험과 많은 내장 기능을 목표로 합니다. 예를 들면:

- 직관적인 페이지 기반 라우팅 시스템 (동적 라우트 지원 포함)
- 페이지별로 지원되는 정적 생성 (SSG) 및 서버 사이드 렌더링 (SSR)에 대한 사전 렌더링
- 더 빠른 페이지 로딩을 위한 자동 코드 분할
- 최적화된 프리페칭을 사용한 클라이언트 측 라우팅
- 내장 CSS 및 Sass 지원 및 CSS-in-JS 라이브러리에 대한 지원
- Fast Refresh 지원 개발 환경
- 서버리스 함수로 API 엔드포인트를 구축하기 위한 API 라우트
- 완전히 확장 가능
Next.js는 전세계의 많은 대형 브랜드를 포함하여 수만 개의 생산 중인 웹사이트와 웹 애플리케이션에서 사용됩니다.

### 이 튜토리얼에 대하여
이 무료 인터랙티브 코스는 Next.js 시작 방법을 안내합니다.

이 튜토리얼에서는 매우 간단한 블로그 앱을 만들며 Next.js 기초를 배웁니다. 최종 결과의 예는 다음과 같습니다:

https://next-learn-starter.vercel.app ([소스](https://github.com/vercel/next-learn/tree/main/basics/demo))

이 튜토리얼은 JavaScript와 React의 기본 지식을 가정합니다. React 코드를 작성해본 적이 없다면, 먼저 공식 React 튜토리얼을 진행해야 합니다.

문서를 찾고 있다면, Next.js 문서를 방문하세요.

## 설정
먼저 개발 환경이 준비되어 있는지 확인합시다.

- Node.js가 설치되어 있지 않다면, 여기서 설치하세요. Node.js 버전 18 이상이 필요합니다.
- 이 튜토리얼에는 자신의 텍스트 에디터와 터미널 앱을 사용할 것입니다.

Windows 사용자라면, Git for Windows를 다운로드하고 포함된 Git Bash를 사용하는 것이 좋습니다. 이는 이 튜토리얼의 UNIX 특정 명령어를 지원합니다. Windows Subsystem for Linux (WSL)도 다른 옵션입니다.

### Next.js 앱 생성
Next.js 앱을 만들려면 터미널을 열고 앱을 생성하려는 디렉토리로 이동한 다음 다음 명령을 실행하세요:
```bash
npx create-next-app@latest \
nextjs-blog --use-npm \
--example \
"https://github.com/vercel/next-learn/tree/main/basics/learn-starter"
```
내부적으로 이는 create-next-app이라는 도구를 사용하여 Next.js 앱을 부트스트랩합니다. --example 플래그를 통해 이 템플릿을 사용합니다.

작동하지 않으면, 이 페이지를 확인하세요.

개발 서버 실행
새로운 디렉토리 nextjs-blog이 생겼습니다. 해당 디렉토리로 이동하려면:
```bash
cd nextjs-blog
```
그런 다음 다음 명령을 실행하세요:
```bash
npm run dev
```
이는 포트 3000에서 Next.js 앱의 "개발 서버"를 시작합니다.

작동하는지 확인해 보세요. 브라우저에서 http://localhost:3000을 엽니다.

## 페이지 편집하기
시작 페이지를 편집해 보겠습니다.

- Next.js 개발 서버가 계속 실행 중인지 확인하세요.
- 텍스트 에디터로 pages/index.js를 엽니다.
- `<h1>` 태그 아래에 “Welcome to”라는 텍스트를 찾아 “Learn”으로 변경하세요.
- 파일을 저장하세요.

파일을 저장하자마자, 브라우저는 새 텍스트로 페이지를 자동으로 업데이트합니다:

![학습하기](https://nextjs.org/static/images/learn/create-nextjs-app/learn-nextjs.png)

Next.js 개발 서버는 Fast Refresh가 활성화되어 있습니다. 파일에 변경 사항을 만들면, Next.js는 브라우저에서 변경 사항을 거의 즉시 자동으로 적용합니다. 새로 고침 필요 없습니다! 이를 통해 앱을 빠르게 반복해서 개선할 수 있습니다.
