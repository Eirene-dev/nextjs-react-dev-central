## 소개
<span className="text-blue-600 font-bold">Next.js</span>를 효과적으로 사용하기 위해서는 
<span className="font-bold underline">JavaScript, React 및 관련 웹 개발 개념</span>에 익숙해야 합니다. 
그러나 JavaScript와 <span className="text-blue-600 font-bold">React</span>는 방대한 주제들입니다. 언제 Next.js를 배우기 시작해야 할지 어떻게 알 수 있을까요?

<span className="font-semibold bg-yellow-100">Next.js 기초 강좌에 오신 것을 환영합니다!</span> 이 초보자 친화적이며 예제 중심의 강좌는 Next.js에 필요한 사전 지식을 안내해 드립니다. 단계별로 간단한 프로젝트를 구축하게 됩니다; JavaScript 애플리케이션으로 시작하여, 이를 React와 Next.js로 마이그레이션하게 됩니다.

각 섹션은 이전 섹션을 기반으로 구축되므로, 이미 알고 있는 내용에 따라 어디서부터 시작할지 선택할 수 있습니다.

이 튜토리얼은 <span className="text-red-600 font-bold">HTML, CSS, JavaScript</span>에 대한 지식을 가정하며, React에 대한 지식은 요구하지 않습니다. 이미 React에 익숙하다면, Next.js 시작하기 섹션으로 건너뛰거나 첫 번째 Next.js 앱을 만들기 시작할 수 있습니다.

## Next.js란 무엇인가요?
Next.js는 빠른 웹 애플리케이션을 만들기 위한 구성 요소를 제공하는 유연한 React 프레임워크입니다.

하지만 이것이 정확히 무엇을 의미하는지? React와 Next.js가 무엇이며 어떻게 도움을 줄 수 있는지에 대해 좀 더 알아보겠습니다.

애플리케이션의 각 부분에서 직접 솔루션을 구축할 것인지 라이브러리와 프레임워크와 같은 다른 도구를 사용할 것인지 결정해야 합니다.

<details className="bg-gray-100 p-2 rounded-md">
  <summary>자세히 알아보기: **인프라 Serverless / CDN / Edge 상세**</summary>

1. **Serverless**:
   - **정의**: Serverless는 서버 인프라 관리의 부담 없이 코드를 실행할 수 있게 해주는 클라우드 컴퓨팅 실행 모델입니다.
   - **작동 방식**: 사용자가 코드를 실행할 때만 클라우드 리소스가 할당되며, 사용이 끝나면 자동으로 해제됩니다.
   - **장점**: 물리적인 서버나 가상 머신을 직접 관리할 필요가 없으며, 사용한 만큼만 비용을 지불하게 됩니다. 또한 자동 확장 기능으로 트래픽에 따라 자동으로 리소스를 조절할 수 있습니다.
   - **예시**: AWS Lambda, Google Cloud Functions, Azure Functions 등이 있습니다.

2. **CDN (Content Delivery Network)**:
   - **정의**: CDN은 전 세계 여러 위치에 데이터 센터를 두고, 사용자에게 최적의 위치에서 콘텐츠를 전송하여 웹사이트의 로딩 시간을 줄이고, 대역폭을 효과적으로 활용하는 시스템입니다.
   - **작동 방식**: 사용자가 웹사이트에 접근하면 CDN은 사용자에게 가장 가까운 서버에서 콘텐츠를 제공합니다. 따라서 데이터의 전송 시간이 단축되고 웹사이트의 반응 속도가 빨라집니다.
   - **장점**: 전세계 어디에서든 빠른 콘텐츠 접근이 가능하며, 웹사이트의 트래픽 부하 분산과 높은 가용성을 보장합니다.
   - **예시**: Cloudflare, AWS CloudFront, Akamai 등이 있습니다.

3. **Edge**:
   - **정의**: Edge 컴퓨팅은 데이터 처리와 서비스 제공을 중앙 데이터 센터가 아닌 사용자에게 가까운 "가장자리"에서 실행하는 방식을 말합니다.
   - **작동 방식**: 필요한 처리를 사용자의 디바이스나 사용자에게 가까운 서버에서 수행함으로써 지연 시간을 줄이고, 대역폭 사용량을 최소화합니다.
   - **장점**: 실시간 처리가 필요한 애플리케이션에 적합하며, 네트워크 지연을 줄일 수 있습니다.
   - **예시**: 자율주행 자동차의 결정 처리, IoT 디바이스에서의 실시간 분석 등에서 활용됩니다.

이 세 가지 기술은 각각 웹 애플리케이션 및 서비스의 성능, 가용성, 그리고 확장성을 향상시키는 데 중요한 역할을 합니다.
</details>

### React란 무엇인가요?
<span className="text-blue-600 font-bold">React</span>는 <span className="font-bold underline">대화식 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리</span>입니다.

![React Introduction](https://nextjs.org/static/images/learn/foundations/user-interface.png)

<span className="text-blue-600 font-bold">사용자 인터페이스(UI)</span>란 화면에서 사용자가 보고 상호작용하는 요소를 의미합니다.

<span className="text-blue-600 font-bold">라이브러리(Library)</span>란, React가 UI 구축에 도움이 되는 함수를 제공하지만 개발자가 그 함수를 어디서 사용할지는 결정하는 것을 의미합니다.

React의 성공의 일부는 응용 프로그램 구축의 다른 측면에 대해 상대적으로 중립적이라는 것입니다.

<details className="bg-gray-100 p-2 rounded-md">
  <summary>자세히 알아보기: **응용 프로그램 구축의 다른 측면에 대해 상대적으로 중립적이라는 의미**</summary>

React는 사용자 인터페이스를 구축하기 위한 JavaScript 라이브러리입니다. React의 핵심 철학 중 하나는 UI 구성 요소를 구축하는 데 집중하면서 다른 여러 응용 프로그램 관련 작업에 대해서는 의견을 가지지 않는 것입니다. 이러한 접근 방식에는 여러 가지 이유가 있습니다:

1. **유연성**: React는 중립적이기 때문에 다양한 백엔드, 플랫폼, 라이브러리 및 도구와 함께 사용할 수 있습니다. 이러한 유연성 덕분에 개발자는 자신의 프로젝트에 가장 적합한 도구와 함께 React를 사용할 수 있습니다.

2. **재사용성**: React 컴포넌트는 재사용이 가능하므로 다양한 환경 및 설정에서 동일한 컴포넌트를 사용할 수 있습니다.

3. **집중**: React 팀은 사용자 인터페이스를 구축하는 데 필요한 핵심 기능에 집중하여 라이브러리를 최적화하고 개선할 수 있습니다. 이렇게 하면 다른 문제에 대한 솔루션을 제공하는 대신 UI 구축에 최적화된 도구를 제공할 수 있습니다.

4. **커뮤니티와 생태계**: React가 다른 응용 프로그램 구축 측면에 대해 중립적이기 때문에 커뮤니티는 다양한 도구, 라이브러리 및 솔루션을 개발하게 되었습니다. 이로 인해 React 생태계는 다양하고 강력한 도구와 라이브러리로 가득 차게 되었습니다.

간단히 말하면, React는 사용자 인터페이스 구축에 집중하면서 다른 응용 프로그램 관련 작업에 대해서는 중립적이라는 접근 방식을 통해 개발자 커뮤니티에 유연성과 선택의 자유를 제공합니다.

</details>


그러나 이것은 또한, React 애플리케이션을 처음부터 완전히 구축하는 데 일부 노력이 필요하다는 것을 의미합니다. 개발자들은 도구를 설정하고 일반 애플리케이션 요구 사항에 대한 솔루션을 다시 발명하는 데 시간을 보내야 합니다.

<details className="bg-gray-100 p-2 rounded-md">
  <summary>자세히 알아보기: **React 앱 구축 시 시간과 노력이 필요한 몇 가지 일반적인 사례**</summary>

React를 사용하여 애플리케이션을 처음부터 구축할 때 시간과 노력이 필요한 몇 가지 일반적인 사례는 다음과 같습니다:

1. **설정 및 보일러플레이트**: 프로젝트를 시작할 때, 웹팩(Webpack), 바벨(Babel), ESLint와 같은 다양한 도구의 설정이 필요합니다. 이러한 도구들은 React 애플리케이션의 성능과 개발 경험을 최적화하는 데 중요하지만 설정과 조정에 시간이 많이 소요될 수 있습니다.

2. **라우팅**: React 자체에는 라우팅 솔루션은 포함되어 있지 않습니다. 따라서 React Router와 같은 서드파티 라이브러리를 도입하고 설정해야 합니다. Next.js에서는 Next.js의 라우팅을 사용 합니다.

3. **상태 관리**: 상태 관리는 복잡한 React 애플리케이션에서 중요한 부분입니다. Redux, MobX, Recoil과 같은 여러 상태 관리 라이브러리 중에서 선택하고 이를 프로젝트에 통합하는 것은 학습 곡선과 설정 시간을 필요로 합니다.

4. **서버 사이드 렌더링 (SSR)**: 초기 페이지 로드 성능을 향상시키려면 SSR이 필요할 수 있습니다. 그러나 React는 기본적으로 클라이언트 사이드 렌더링을 사용하므로 SSR을 구현하려면 추가적인 설정과 로직이 필요합니다.

5. **API 호출 및 데이터 가져오기**: React 애플리케이션에서 데이터를 가져오거나 API와 통신하려면 Axios나 Fetch와 같은 도구나 메서드를 사용해야 합니다. 각 도구에는 자체적인 설정과 사용 방법이 있습니다.

6. **옵티마이제이션**: React 애플리케이션의 성능을 최적화하려면 메모이제이션, 코드 스플리팅, 레이지 로딩과 같은 다양한 전략을 적용해야 할 수 있습니다.

이러한 과정들은 React 애플리케이션을 처음부터 완전히 구축하는 데 필요한 단계들입니다. 각 단계는 자체적인 학습 곡선과 설정 시간을 가지고 있으므로 개발자는 시간과 노력을 투자해야 합니다.
</details>

### Next.js란 무엇인가요?
<span className="text-blue-600 font-bold">Next.js</span>는
 <span className="font-bold underline">웹 애플리케이션을 만들기 위한 구성 요소를 제공하는 React 프레임워크</span>입니다.

<span className="text-blue-600 font-bold">프레임워크</span>라는 것은 Next.js가 React에 필요한 도구 및 설정을 처리하며 애플리케이션에 대한 추가 구조, 기능 및 최적화를 제공한다는 것을 의미합니다.

![Next.js Introduction](https://nextjs.org/static/images/learn/foundations/next-app.png)

React를 사용하여 UI를 구축한 다음, 라우팅, 데이터 가져오기, 통합과 같은 일반적인 애플리케이션 요구 사항을 해결하기 위해 Next.js의 기능을 점진적으로 적용할 수 있습니다. 이 모든 것은 개발자와 최종 사용자의 경험을 향상시키면서도요.

<details className="bg-gray-100 p-2 rounded-md">
  <summary>자세히 알아보기: **React와 Next.js의 상호 관계**</summary>

React는 사용자 인터페이스를 구축하기 위한 자바스크립트 라이브러리입니다. 이는 컴포넌트 기반의 UI를 만들기 위한 기본 구성 요소를 제공합니다. 반면, Next.js는 React를 기반으로 하는 프레임워크입니다. 이는 React의 컴포넌트 구조를 유지하면서 웹 애플리케이션을 빌드, 최적화, 배포하는 과정을 단순화하고 향상시키기 위한 추가 기능과 도구를 제공합니다.
</details>

<Callout type="warning">
  <div className="flex items-center">
    <Info />
    <span className="ml-2">**Next.js를 사용할 때 React 개발자가 주의해야 할 사항**</span>
  </div>

Next.js는 React를 기반으로 한 프레임워크이지만, 몇 가지 주요 차이점과 주의해야 할 사항이 있습니다. 

1. **라이프사이클 메서드**: Next.js의 서버 사이드 렌더링을 이용할 때, 일부 React 라이프사이클 메서드들은 클라이언트 사이드에서만 동작합니다. 예를 들어, `componentDidMount`는 서버 사이드에서는 호출되지 않습니다.

2. **스타일링**: Next.js는 기본적으로 `styled-jsx`를 내장하고 있습니다. 이를 사용해 컴포넌트 수준의 스타일링을 할 수 있으나, 다른 스타일링 라이브러리나 방법을 사용하려면 추가 설정이 필요할 수 있습니다.

3. **데이터 페칭**: 기존 React에서의 데이터 페칭과는 다르게, Next.js의 페이지 라우터에서는 `getServerSideProps`, `getStaticProps`, `getStaticPaths`와 같은 특별한 함수를 사용하여 데이터를 사전에 가져올 수 있습니다.

4. **커스텀 `_app` 및 `_document`**: 전역 애플리케이션 로직 및 사용자 정의 문서를 위한 특별한 파일입니다. React의 전통적인 패턴과는 약간 다를 수 있으므로 이를 이해하고 활용하는 것이 중요합니다.

이러한 차이점과 특성을 숙지하면, React 기반의 지식을 가진 개발자도 Next.js를 효과적으로 사용할 수 있습니다.
</Callout>

Next.js는 원래 페이지 기반의 라우팅 방식인 페이지 라우터를 사용하였으나, 최근에는 React 서버/클라이언트 컴포넌트를 기반으로 앱 라우터 방식을 채택하며 이를 권장하고 있습니다. 이 전환에 따라 라우팅과 데이터 페칭 등의 주요 메커니즘이 변경되었습니다.
