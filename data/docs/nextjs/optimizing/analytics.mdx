[Next.js Speed Insights](https://nextjs.org/analytics)는 다양한 지표를 사용하여 페이지의 성능을 분석하고 측정할 수 있게 해줍니다.

[Vercel 배포](https://vercel.com/docs/concepts/speed-insights?utm_source=next-site&utm_medium=docs&utm_campaign=next-website)에서 설정 없이 [Real Experience Score](https://vercel.com/docs/concepts/speed-insights#core-web-vitals-explained?utm_source=next-site&utm_medium=docs&utm_campaign=next-website)를 수집하기 시작할 수 있습니다.

이 문서의 나머지 부분은 Next.js Speed Insights가 사용하는 내장된 중계기에 대해 설명합니다.

## 웹 바이탈

[Web Vitals](https://web.dev/vitals/)는 웹 페이지의 사용자 경험을 포착하려는 목표로 유용한 지표들의 집합입니다. 다음 웹 바이탈들이 모두 포함됩니다:

- [첫 번째 바이트까지의 시간](https://developer.mozilla.org/docs/Glossary/Time_to_first_byte) (TTFB)
- [첫 컨텐츠 그리기](https://developer.mozilla.org/docs/Glossary/First_contentful_paint) (FCP)
- [가장 큰 컨텐츠 그리기](https://web.dev/lcp/) (LCP)
- [첫 입력 지연](https://web.dev/fid/) (FID)
- [누적 레이아웃 이동](https://web.dev/cls/) (CLS)
- [다음 그림까지의 상호작용](https://web.dev/inp/) (INP) _(실험적)_
