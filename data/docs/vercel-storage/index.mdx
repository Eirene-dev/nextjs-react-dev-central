## Vercel 스토리지

Vercel의 서버리스 스토리지 제품군으로 키-값 데이터, 트랜잭션 데이터, 대용량 파일 등을 저장합니다. 이 페이지는 사용 사례에 맞는 적절한 스토리지 제품을 선택하는 데 도움을 줍니다.

### 스토리지 제품 선택하기

적절한 스토리지 솔루션을 선택하는 것은 지연 시간, 내구성, 일관성 등 다양한 요소를 고려해야 합니다. 각 스토리지 옵션의 장점을 아래 표에서 비교할 수 있습니다.

| 제품       | 읽기       | 쓰기         | 사용 사례                          | 
|----------|----------|------------|---------------------------------|
| KV       | 빠름        | 밀리초 단위     | 키/값 및 JSON 데이터                 | 
| Postgres | 빠름        | 밀리초 단위     | 구조화된 관계형 데이터                 | 
| Blob     | 빠름        | 밀리초 단위     | 대용량, 콘텐츠 주소 지정 파일 ("블롭")   | 
| Edge Config | 초고속    | 초 단위       | 런타임 구성 (예: 기능 플래그)         |

스토리지 제품에 대한 최선의 활용 방법을 알아보려면 스토리지 제품에 대한 최적의 사용 사례 섹션을 참조하세요.

### Vercel KV

Vercel KV는 내구성 있는 서버리스 Redis 스토리지입니다. Redis API를 사용하고자 하며, 빠른 키-값 데이터 접근이 필요하거나 여러 지역에 데이터를 복제해야 하는 경우 Vercel KV를 사용하는 것이 좋습니다.

### Vercel Postgres

Vercel Postgres는 Vercel 함수와 프론트엔드 프레임워크와 통합되도록 설계된 서버리스 PostgreSQL 데이터베이스입니다. 복잡하고 트랜잭션 데이터 관리가 필요하거나 다양한 데이터 형식 또는 사용자 정의 데이터 타입을 다루어야 할 때 Vercel Postgres를 사용하는 것이 좋습니다.

### Vercel Blob

Vercel Blob은 이미지, 비디오 및 기타 파일을 최적화하여 저장합니다. 사용자 아바타 또는 제품 이미지, 사용자 생성 비디오 콘텐츠를 저장해야 하는 경우 Vercel Blob을 사용하는 것이 좋습니다.

### Edge Config

Edge Config는 전역 데이터 스토어로, 외부 데이터베이스나 상위 서버를 조회하지 않고도 엣지에서 데이터를 읽을 수 있습니다. 기능 플래그를 저장하거나 자주 변경되지 않지만 자주 읽히는 데이터를 저장해야 할 경우 Edge Config를 사용하는 것이 좋습니다.

## 최적의 사용 사례

데이터를 함수에 가까이 위치시켜 낮은 지연 시간으로 응답을 보장합니다. 데이터베이스를 함수와 같은 지역에 배치하고, Vercel KV 및 복제된 지역의 경우, 함수와 같은 지역에 스토어를 배치합니다. 데이터베이스를 계정 간에 이전할 수도 있습니다.
