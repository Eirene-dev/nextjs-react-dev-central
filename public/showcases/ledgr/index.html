스트라이프 랜딩의 디자인 문법(그라디언트 메시·코드와 마케팅 공존·로고월·문서형 푸터)을 연구한 가상 브랜드 데모 — 실제 로고·제품·카피 미사용

payments_api/v3

결제를, 몇 줄의 코드로.

Ledgr는 개발자를 위한 결제 인프라입니다. 카드·계좌이체·정산까지, 하나의 API로.

문서 보기키 발급

읽으면 그대로 동작하는 API

아래로 읽어 내려가면, 코드가 단계를 따라옵니다. 결제부터 환불까지 네 줄로.

01 · CHARGES

결제 생성

한 번의 호출로 카드·계좌이체 결제를 만듭니다. 멱등성 키로 중복도 안전합니다.

02 · WEBHOOKS

웹훅 서명 검증

수신 이벤트의 서명을 내장 함수로 검증해 위·변조를 차단합니다.

03 · PAYOUTS

정산 조회

정산 상태와 도착 예정일을 정산 ID로 조회합니다.

04 · REFUNDS

환불

전액·부분 환불을 한 줄로. 원거래에 자동으로 연결됩니다.

필요한 건, 다 있습니다

결제부터 사기 방지까지. 하나의 API로 끝내는 결제 인프라.

결제

카드·계좌이체·간편결제를 한 번의 호출로 처리합니다.

정산

도착 예정일·상태를 추적하고 자동으로 입금합니다.

청구

구독·인보이스·미터링 과금을 코드로 구성합니다.

사기 방지

위험 점수와 규칙으로 의심 거래를 자동 차단합니다.

웹훅

이벤트를 실시간으로 받고, 서명 검증이 내장돼 있습니다.

멱등성·테스트키

안전한 재시도와 운영·테스트 키 분리를 기본 제공합니다.

요청 한 번이 정산까지

결제 요청부터 정산·웹훅 알림까지, Ledgr가 흐름을 잇습니다.

웹훅, 검증까지 한 줄

이벤트 페이로드를 받고, 내장 함수로 서명을 검증해 위·변조를 막습니다.

POST /webhooks · event payload
{
  "id": "evt_1NxY",
  "type": "charge.succeeded",
  "data": {
    "amount": 12000,
    "currency": "krw"
  }
}
verify.js · 서명 검증
// 서명 검증 후 안전하게 처리
const event = ledgr.webhooks.constructEvent(
  payload, sig, endpointSecret,
);
if (event.type === 'charge.succeeded') {
  fulfill(event.data);
}
